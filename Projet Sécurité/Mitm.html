Mise en situation

Pour simuler notre attaque Man in the middle (Mitm), nous utilisons un serveur ftp (proftpd), installé sur notre MachineServeur (Linux). Celui a été configuré de manière à permettre la connexion anonyme ainsi que des droits complets à celle-ci. Nous chercherons à déposer un fichier sur ce serveur, à partir de la MachineClient (Windows). La connexion ainsi que ce fichier seront interceptés par une troisième machine (attaquant), nommée MachineCapture (Mitm).

Victimes :

MachineServeur : 192.168.4.206

MachineClient : 192.168.4.204








Affichage console et connexion ftp


On ouvre une console CMD sous Windows :

Tapez la commande pour se connecter au serveur ftp (MachineServeur : 192.168.4.206) :

ftp 192.168.4.206






Renseignement de l'utilisateur

Renseignez l’utilisateur (ftp) :

ftp






Renseignement du mot de passe

Renseignez le mot de passe (motdepasse) :

motdepasse






Visualisation contenu serveur

Cherchez à savoir ce qu’il y a actuellement sur le serveur ftp :

ls






Déconnexion

Déconnectez-vous du serveur ftp :

bye






Création de fichier (Etape 1)


On veut créer un fichier transfert.txt (pour l’envoyer ensuite sur le serveur).

Déplacez-vous d’abord dans le dossier voulu, ici Documents :

cd Documents






Création de fichier (Etape 2)


On veut créer un fichier transfert.txt (pour l’envoyer ensuite sur le serveur).

Créez maintenant le fichier "fichier.txt" contenant le texte suivant "texte dans le fichier de transfert" :

echo texte dans le fichier de transfert > fichier.txt
